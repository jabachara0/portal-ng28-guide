# portal-ng28-guide

本仓库用于归档和发布若干独立 HTML 页面。这些页面以通用结构组织，不针对特定域名或具体网站，仅作为页面集合的存放与分发点。

## 项目简介

portal-ng28-guide 是一个静态页面归档仓库。所有页面均为独立的 HTML 文件，不依赖后端服务或数据库。仓库的主要目的是提供一份可浏览、可引用的页面集合，方便查阅与维护。

## 目录结构

```
portal-ng28-guide/
├── index.html          # 仓库入口页面（导航或概览）
├── pages/              # 页面存放目录
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
├── assets/             # 公共资源目录（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

页面文件均置于 `pages/` 目录下，`index.html` 可作为入口或索引。`assets/` 目录用于存放页面共用的样式、脚本或图片资源。

## 页面归档说明

- 每个页面均为独立、自包含的 HTML 文件。
- 页面之间无强制依赖关系，可单独打开或引用。
- 归档内容不指向任何特定域名或具体网站，仅作为静态页面本身存在。
- 页面命名遵循 `page-编号.html` 的格式，编号无特殊含义，仅用于区分。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接使用浏览器打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可将仓库部署到任意静态托管服务（如 GitHub Pages），通过 URL 访问页面。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的导航（如有）。
- 修改页面：直接编辑对应 HTML 文件，保持文件结构与命名规范。
- 删除页面：移除对应文件，并同步更新 `index.html`。
- 资源文件：如需添加公共资源，请放入 `assets/` 对应子目录，并在页面中正确引用路径。
